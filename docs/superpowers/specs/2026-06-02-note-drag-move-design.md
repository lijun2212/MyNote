## 1. 目标与范围

本次功能允许用户在左侧文件树中直接拖拽一篇文章，并将其放入某个笔记本或笔记本内部的任意子目录。

目标是让文件树具备最直接的整理能力，不再要求用户通过重建笔记或手工改路径来完成归档。

本次范围只覆盖“单篇文章移动”。不包含目录拖拽、不包含批量拖拽、不包含覆盖确认弹窗。

## 2. 用户交互

### 2.1 可拖拽对象

- 文件树中的文章节点支持拖拽。
- 目录节点不支持被拖拽。

### 2.2 可放置目标

- `notes` 下的顶层笔记本目录可以作为放置目标。
- 笔记本内部的任意子目录也可以作为放置目标。
- 普通文章节点不能作为放置目标。
- “未归档”只是前端视图分组，不是物理目录，不能作为放置目标。

### 2.3 放置反馈

- 当正在拖拽文章且鼠标进入可放置目录时，目标目录显示明确的悬停高亮。
- 当鼠标离开目标目录时，高亮立即消失。
- 放置完成后刷新文件树。

### 2.4 编辑区行为

- 如果被拖拽的是当前正在打开的文章，移动成功后只刷新文件树。
- 编辑区保持现状，不自动跳转到新路径，不主动关闭，不自动切换选中项。

## 3. 移动规则

### 3.1 基本规则

- 拖动源必须是单篇文章路径。
- 放置目标必须是 `notes/...` 下的真实目录路径。
- 如果源文章已经位于目标目录中，则本次放置视为 no-op，不执行移动。

### 3.2 同名冲突规则

- 如果目标目录存在同名文件，系统自动重命名。
- 重命名规则为在文件名后追加递增后缀，例如：
  - `合同审查.md`
  - `合同审查-1.md`
  - `合同审查-2.md`
- 自动重命名发生在后端，前端不自行推导最终路径。

### 3.3 索引更新规则

- 文件移动成功后，后端必须更新数据库索引，使新路径立即可见。
- 旧路径不应继续出现在文件树或索引结果中。
- 返回值必须包含最终实际路径，供前端后续刷新或扩展使用。

## 4. 前后端设计

### 4.1 后端

新增 `move_note` command，输入为：

- 源文章相对路径 `sourcePath`
- 目标目录相对路径 `targetDirectory`

后端职责：

- 校验源路径是存在的文章文件。
- 校验目标路径是 `notes` 下的合法目录。
- 执行物理文件移动。
- 处理同名冲突并生成最终文件名。
- 重新索引移动后的文章。
- 清理旧路径索引记录。
- 返回最终文章对象或至少返回最终相对路径。

推荐返回完整 `Note`，这样前端后续若要同步选中状态或编辑区，不需要再补一次查询。

### 4.2 前端

文件树节点扩展两类能力：

- 文件节点：`dragstart`
- 目录节点：`dragenter` / `dragover` / `dragleave` / `drop`

前端职责：

- 只允许文章节点发起拖拽。
- 只允许真实目录节点接收放置。
- 明确排除“未归档”视图分组节点。
- drop 后调用 `moveNote` API。
- 成功后调用 `refreshTree()`。
- 当前版本不修改编辑区当前打开内容。

为了避免把所有逻辑塞进单个组件，建议新增一个轻量判断层，用于回答：

- 某节点是否可拖拽
- 某节点是否可放置
- 某放置目标对应的真实目录路径是什么

## 5. 错误处理

- 如果源文章不存在，后端返回 `NotFound`。
- 如果目标目录不存在或不是合法目录，后端返回 `InvalidInput`。
- 如果移动过程中出现 IO 异常，后端返回 `Io`。
- 前端当前阶段只需要记录错误并停止本次拖拽结果，不引入复杂提示系统。

## 6. 测试与验证

### 6.1 Rust 测试

至少覆盖以下场景：

- 将文章移动到顶层笔记本目录。
- 将文章移动到笔记本内部子目录。
- 目标目录同名时自动生成 `-1`、`-2` 后缀。
- 源文章已在目标目录时 no-op。
- 移动后新路径被索引，旧路径不再可见。

### 6.2 前端测试

至少覆盖以下场景：

- 文件节点可拖拽，目录节点不可拖拽。
- 顶层笔记本和内部目录可接收放置。
- “未归档”不能接收放置。
- drop 后调用 `moveNote` 并刷新文件树。

## 7. 非目标

以下内容不在本次实现范围内：

- 拖拽目录移动整个目录树
- 批量拖拽多篇文章
- 拖拽动画和复杂视觉反馈
- 移动成功后的编辑区自动跳转
- 冲突时弹出选择覆盖/重命名/取消对话框
