## 1. 背景

当前侧边栏为笔记标题 hover 展示摘要时，主要问题不在“看不到摘要”，而在“摘要遮挡目录树内容，影响扫描目录与文章标题”。

用户已明确要求：

1. 保留悬浮气泡，而不是改成固定面板或彻底移除。
2. 气泡应改为半透明，使用户能透过气泡看到后方的目录和文章标题。
3. 视觉上应降低遮挡感，而不是继续扩大或强化摘要面板的存在感。

## 2. 目标

1. 保留 hover 即得摘要的轻量预览能力。
2. 明显降低气泡对目录树内容的遮挡感。
3. 控制气泡体积，避免摘要一长就成为大面积浮层。
4. 不改变当前“摘要来源于当前笔记 summary 字段”的数据模型。

## 3. 非目标

本轮不处理以下内容：

1. 不改为固定摘要栏、底部摘要栏或右侧独立摘要面板。
2. 不改变摘要的生成、保存、索引策略。
3. 不引入点击展开、固定钉住、手动收起等新交互。
4. 不重构整个文件树布局。

## 4. 方案结论

采用“半透明悬浮气泡 + 严格尺寸约束”的方案。

结论如下：

1. 仍然使用 hover 悬浮气泡承载摘要。
2. 气泡背景改为半透明浅色，允许后方目录文字隐约可见。
3. 同时保留轻量边框与阴影，确保气泡文字本身仍可读。
4. 气泡尺寸必须受限，不能因为摘要过长而扩张为大面积遮挡层。

这是在“保留当前交互模式”和“降低视觉干扰”之间的折中方案。

## 5. 交互规则

### 5.1 出现规则

1. 用户 hover 某篇笔记标题时，若该笔记存在摘要，则显示悬浮气泡。
2. hover 离开后，气泡消失。
3. 气泡保持 `pointer-events: none`，不拦截当前目录树交互。

### 5.2 不改变的信息密度原则

目录树的主任务仍是浏览与定位笔记，摘要只是辅助预览。因此气泡必须服从目录树，而不能在视觉上压过目录树。

## 6. 视觉规则

### 6.1 背景

气泡背景采用半透明暖色系，例如接近：

- `rgba(255, 251, 235, 0.7x)`

要求：

1. 后方目录文字仍能隐约可见。
2. 前景摘要文字保持足够对比度，不出现难以阅读的叠影。

### 6.2 边框与阴影

1. 边框保留，但弱化强度。
2. 阴影保留，但弱于当前版本，避免形成厚重浮层感。

### 6.3 文本

1. 支持自然换行。
2. 支持较长中文摘要的连续阅读。
3. 不使用额外标题、图标或操作按钮，避免增加视觉重量。

## 7. 尺寸与边界规则

### 7.1 宽度

1. 气泡宽度应固定在较克制的范围内。
2. 不因摘要变长而横向扩张。

### 7.2 高度

1. 气泡最大高度受限。
2. 超出后允许内部滚动，而不是继续覆盖更多目录内容。

### 7.3 位置

1. 气泡仍沿用当前右侧浮出策略。
2. 当接近可滚动区域底部时，需要自动上移，避免被容器裁剪。

## 8. 验收标准

满足以下条件即可验收：

1. hover 某篇有摘要的笔记时，仍能出现摘要气泡。
2. 气泡覆盖到目录树时，用户仍能肉眼辨认后方的文章标题和目录文字。
3. 长摘要不会让气泡无限增高或大面积遮挡目录树。
4. 目录树点击、hover、拖拽等原有交互不被气泡拦截。
5. 无摘要笔记不显示气泡。

## 9. 风险与处理

### 9.1 风险：半透明后前后文字叠影过重

处理：

1. 背景透明度不要过低。
2. 保留轻边框和轻阴影，建立前后层次。

### 9.2 风险：长摘要虽然半透明，但仍然占据大片面积

处理：

1. 严格控制最大宽高。
2. 超出后内部滚动。

### 9.3 风险：为了“看见后面”而牺牲前景可读性

处理：

1. 优先保证摘要文字可读。
2. “后面隐约可见”即可，不追求完全透明。
