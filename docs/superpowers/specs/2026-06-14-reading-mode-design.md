## 1. 背景与问题定义

当前 MyNote 中间工作区实际上只有两种可表达形态：

1. 双列模式：编辑区 + 预览区。
2. 单列编辑模式：仅编辑区。

用户新增的需求不是简单的“隐藏/显示预览”，而是需要第三种独立工作形态：纯阅读模式，即整个中间区只展示预览结果，不展示编辑面板。

在已有交互中，第三个顶部按钮使用“显示/隐藏预览”的二元语义。这种语义无法自然承载以下三种状态：

1. 双列模式。
2. 纯阅读模式。
3. 纯写作模式。

如果继续沿用二元切换，不仅按钮语义会冲突，也会导致打开笔记时的默认行为、状态恢复、测试语义都变得含糊。因此需要把中间工作区从二态模型升级为三态模型，并重新定义顶部按钮职责。

## 2. 目标与非目标

### 2.1 目标

- 支持纯阅读模式：中间区仅显示预览。
- 支持纯写作模式：中间区仅显示编辑器。
- 保持双列模式为默认工作形态。
- 在单列模式之间提供低成本切换入口。
- 在纯阅读/纯写作下提供显式的“返回双列”入口。
- 切回双列时恢复进入单列前的分栏比例。
- 打开新笔记时恢复为双列模式，而不是延续上一篇笔记的单列状态。

### 2.2 非目标

- 本轮不扩展菜单项与快捷键。
- 本轮不新增跨会话恢复“纯阅读/纯写作”状态。
- 本轮不重做顶部工具栏整体视觉结构。
- 本轮不改动投影预览的独立窗口设计。

## 3. 方案对比与结论

### 3.1 方案 A：单按钮三态循环

做法：用一个按钮在“双列 -> 纯阅读 -> 纯写作”之间轮转。

优点：

- 控件数量最少。

缺点：

- 语义不直观。
- 无法快速返回双列。
- 用户很难预判下一次点击会落到哪个状态。

结论：不采用。

### 3.2 方案 B：第三按钮负责单列切换，新增独立“返回双列”图标按钮

做法：

- 顶部前三个主按钮保留结构。
- 第三个按钮改成“阅读模式 / 写作模式”切换按钮。
- 在纯阅读或纯写作时，在其右侧出现一个仅图标的“返回双列”按钮。

优点：

- 语义清晰。
- 单列与双列职责分离。
- 用户随时能一跳回到默认双列。

缺点：

- 单列模式下工具栏会从 3 个主按钮临时扩成 4 个控件。

结论：采用。

### 3.3 方案 C：把“返回双列”放到悬浮位置

优点：

- 顶部工具栏结构更稳定。

缺点：

- 入口漂浮，不如工具栏稳定。
- 发现成本更高。

结论：不采用。

## 4. 状态模型设计

中间工作区升级为三态视图模型：

1. `split`
表示双列模式：左侧编辑区，右侧预览区。

2. `preview`
表示纯阅读模式：只显示预览区。

3. `editor`
表示纯写作模式：只显示编辑区。

为降低改动风险，本轮不立即硬删现有 `showPreview` 布尔值，而是让它退化为兼容派生语义：

- `showPreview = viewMode !== "editor"`

这样现有依赖“有没有预览”的部分可以逐步迁移，而顶部工作区和主状态流直接基于 `viewMode` 决策。

## 5. 交互与界面设计

### 5.1 顶部按钮布局

保持现有前三个主按钮的整体位置：

1. `展开摘要`
2. `开启投影`
3. 单列模式切换按钮

在第 3 个按钮右侧新增一个仅图标按钮：

- 语义：返回双列
- 显示条件：仅在 `preview` 或 `editor` 下显示
- 默认双列下完全不显示

### 5.2 第三个按钮文案与行为

在不同状态下，第 3 个按钮文案如下：

1. `split` 时：显示 `阅读模式`
点击后进入 `preview`。

2. `preview` 时：显示 `写作模式`
点击后进入 `editor`。

3. `editor` 时：显示 `阅读模式`
点击后进入 `preview`。

这个按钮只负责在两种单列模式之间切换，不承担返回双列的职责。

### 5.3 返回双列按钮行为

返回双列按钮只做一件事：

- 在 `preview` 或 `editor` 下点击后，直接回到 `split`

它不显示文字，仅显示图标，并沿用当前工具按钮的浅蓝 hover 风格，避免引入新的视觉语言。

### 5.4 中间区渲染规则

1. `split`

- 显示编辑区
- 显示中间分隔条
- 显示预览区

2. `preview`

- 仅显示预览区
- 不渲染编辑区
- 不渲染中间分隔条

3. `editor`

- 仅显示编辑区
- 不渲染预览区
- 不渲染中间分隔条

### 5.5 分栏比例恢复规则

用户从 `split` 切到 `preview` 或 `editor` 时，不覆盖当前已保存的分栏比例。

用户从单列模式点击“返回双列”后，恢复到进入单列前的分栏比例，而不是重置到新的值。默认值仍为 50%。

## 6. 状态持久化与兼容策略

### 6.1 store 层

新增 `viewMode` 作为中间工作区的直接状态源。

现有与编辑模式相关的派生接口可短期保留，但内部映射为：

- `split` -> 分栏
- `editor` -> 仅编辑
- `preview` -> 纯阅读

### 6.2 打开新笔记的默认行为

虽然用户可以临时进入纯阅读或纯写作，但“打开一篇新笔记”的默认行为仍然恢复为 `split`。

这是本设计的明确产品约束：

- 默认工作形态始终是双列
- 阅读/写作模式是临时聚焦态，不跨笔记延续

### 6.3 持久化范围

本轮不持久化 `preview` / `editor` 这种单列状态。

保留现有分栏比例 localStorage 逻辑，但不新增“跨会话记住阅读模式”的行为。这样可以避免用户下次进入应用时误以为布局异常。

## 7. 实现边界

本轮实现边界收敛为：

1. store 新增三态 `viewMode`
2. EditorWorkspace 顶部按钮与中间区三态渲染
3. 返回双列图标按钮
4. 打开新笔记恢复 `split`
5. 单列进入/退出时恢复既有分栏比例

本轮明确不做：

1. 菜单项改造
2. 快捷键改造
3. 右键菜单入口补充
4. 跨会话持久化单列模式

## 8. 测试策略

### 8.1 store 测试

- `viewMode` 三态切换正确
- 兼容派生值与旧接口映射正确

### 8.2 EditorWorkspace 渲染测试

- `split` 时显示编辑器、预览和分隔条
- `preview` 时仅显示预览
- `editor` 时仅显示编辑器
- 第三个按钮文案随状态变化正确
- 返回双列按钮只在单列模式出现

### 8.3 交互测试

- `split -> preview`
- `preview -> editor`
- `editor -> preview`
- `preview/editor -> split`

### 8.4 回归测试

- 打开新笔记时恢复 `split`
- 双列比例在退出单列后恢复
- 长标题下顶部工具栏与中间区布局不被撑破

## 9. 验收标准

满足以下条件即视为验收通过：

1. 默认打开任意笔记时，中间区为双列模式。
2. 点击“阅读模式”后，中间区仅显示预览。
3. 纯阅读模式下，第 3 个按钮显示“写作模式”。
4. 点击“写作模式”后，中间区仅显示编辑器。
5. 纯阅读或纯写作下，返回双列图标按钮出现并可一键恢复双列。
6. 回到双列后，恢复进入单列前的分栏比例。
7. 打开新笔记时，不延续上一篇笔记的纯阅读/纯写作状态。
8. 三种模式下顶部工具栏不出现布局挤压，长标题也不会把按钮或预览区挤出可视区域。

## 10. 风险与后续事项

### 10.1 主要风险

- 旧逻辑大量依赖 `showPreview`，迁移不彻底时可能出现局部判断混杂。
- 现有测试默认把中间工作区理解为二态，需要逐步校正测试语义。

### 10.2 后续事项

- 如本轮落地稳定，可在下一轮补充菜单入口与快捷键。
- 如用户后续需要，可再讨论是否跨会话记住阅读/写作模式。
- 如投影预览与阅读模式后续要协同，可再补“阅读模式下自动隐藏无关提示文案”的视觉优化。
