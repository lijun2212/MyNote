# 背景

当前欢迎页仍然偏脚手架默认风格：结构简单，但缺少产品气质，也没有把 MyNote 的核心价值讲清楚。用户希望它体现“简单易用的个人笔记本”这一身份，不像复杂的知识管理系统首页，而像一本熟悉、亲切、可以长期陪伴自己的笔记本。

同时，这个入口不能只停留在“记录工具”层面。它还需要自然带出更深一层的价值：记录日常、积累知识、形成脉络、内化能力。表达上要更温暖、更有人情味，但不能写成过度抒情的长篇欢迎辞。

# 目标

本次欢迎页改版目标如下：

- 让用户第一眼感受到 MyNote 是“写给自己的笔记本”，而不是生硬的工具启动页。
- 用温暖、亲切、带陪伴感的中文文案传达“熟悉、自然、容易开始”的产品气质。
- 在不增加学习成本的前提下，补充第二层产品价值：从记录到整理，再到知识沉淀与能力内化。
- 保持页面清爽安静，避免做成营销落地页、SaaS 登录页或复杂知识图谱首页。

# 设计原则

## 1. 先陪伴，后能力

欢迎页的第一层价值是“陪你记录”，第二层价值才是“帮你沉淀”。用户不应该在首页先被“知识图谱”“能力系统”这类抽象词汇压住，而应该先感到熟悉、自然、好开始。

## 2. 先日常，后方法

文案要先写“日子、想法、成长、片段”，再延伸到“知识、整理、脉络”。这样更符合个人笔记本的使用直觉，也更贴近用户描述的“像中学时整理学习笔记一样熟悉”。

## 3. 温暖但不滥情

整体语气应亲切、柔和、有一点感情，但不写成散文式长文，也不做廉价煽情。页面仍然要服务于启动动作，不能让用户读完一大段后才找到按钮。

## 4. 简单但不空

页面要保持启动入口的轻量感，但不能只有一个标题和两个按钮。需要有足够的叙事，让用户明白这款产品为何存在。

# 信息架构

欢迎页采用左右双区布局，建议桌面端比例约为 6:5。

左侧为主叙事区，承载品牌身份、主标题、核心说明、价值短句与主次按钮。

右侧为轻场景区，负责营造“熟悉的笔记本”氛围，并用极轻量的视觉路径暗示“记录 -> 整理 -> 脉络”的价值延伸。

整体结构如下：

1. 小标签：写给自己的笔记本
2. 主标题：把日子、想法和成长，慢慢记下来
3. 核心说明：两段短文，表达熟悉感、陪伴感与长期积累
4. 三条价值短句：强调记录、找回、沉淀
5. 次级补充：点明“能力内化”是第二层价值
6. 主次按钮：新建知识库 / 打开知识库
7. 右侧场景：纸页、卡片、轻连接路径

# 文案方案

## 固定文案

### 小标签

写给自己的笔记本

### 主标题

把日子、想法和成长，慢慢记下来

### 核心说明

就像学生时代整理课本和笔记那样，熟悉、自然，不需要重新学习怎么开始。

想到什么，就先写下来；过些时候再回来看，它们会一点点连成你的日常、你的知识，也连成你自己。

### 三条价值短句

- 随手写下此刻的生活与念头
- 随时找回那些重要的片段
- 让零散记录慢慢沉淀成自己的脉络

### 次级补充

MyNote 不只是帮你保存内容。

它也陪你把每天的记录、一路的学习和长久的积累，慢慢整理成更清楚的理解，内化成真正属于你的能力。

### 按钮文案

- 主按钮：新建知识库
- 次按钮：打开知识库

## 文案气质要求

- 使用第二人称和偏口语化表达，但避免网络化和过度热情。
- 避免“效率翻倍”“构建第二大脑”“知识资产化”等过强工具化或营销化措辞。
- 避免把“图谱”直接放进主标题或首屏第一句；如果要出现，也只能在次级说明或视觉暗示中轻轻带出。

# 视觉与氛围

## 背景

背景不使用纯白，也不使用强渐变。采用暖米白纸面色作为主底色，叠加非常轻的纸张纹理、柔和阴影或晨光般的明暗变化，营造“翻开一页笔记”的感觉。

## 色彩

- 主文字：接近墨色的深灰，而不是纯黑。
- 点缀色：偏植物感的绿色，与现有 icon 气质一致。
- 辅助色：米黄、雾灰、浅暖白。

不使用高饱和蓝紫系科技色，也不做亮色营销按钮风格。

## 版式

左侧文案区不放进厚重卡片中，直接在背景上组织层级，让页面更像自然展开的一页内容，而不是 SaaS 登录面板。

右侧场景区采用“纸页 + 卡片 + 连接线”的轻拼贴方式，而不是复杂插画或写实桌面图。

## 右侧场景元素

右侧建议包含以下元素：

- 一张主笔记页：像刚写下的一页记录，有标题、短段落和一处轻手写感标记。
- 两张辅助卡片：一张偏日常片段，一张偏知识摘记。
- 一条很淡的路径或节点线：暗示从“记下”到“整理”再到“脉络”。

该路径只做寓意，不做复杂知识图谱视觉，以免抢走首页亲切感。

# 交互与状态

## 动效

欢迎页只需要轻量动效：

- 主标题和副文案轻微上浮显现
- 右侧纸页和卡片错落出现
- 按钮 hover 做小幅明度变化和浅阴影反馈

禁止使用跳跃、旋转、强缩放或明显科技感粒子效果。

## 响应式处理

桌面端保持左右双区。

窄屏或较小窗口下，右侧场景区移动到主内容下方，整体变为纵向布局。文案层级与按钮顺序保持不变，确保“先读文案，再做选择”。

# 实现边界

本次设计聚焦欢迎页入口体验，不包括以下改动：

- 主应用壳层和进入知识库后的整体视觉重做
- 首页按钮背后的业务流程修改
- 系统文件选择器文案和 macOS 原生面板本地化问题
- 搜索、笔记树、编辑器、图谱等后续页面信息架构调整

欢迎页改版应尽量复用现有 `WelcomeScreen` 的行为逻辑，只重构文案、布局与视觉表达。

# 验收标准

欢迎页改版完成后，应满足以下标准：

- 页面第一眼传达出“个人笔记本”的熟悉感与陪伴感。
- 文案语气温暖亲切，较当前版本明显更有情感，但不显得拖沓或煽情。
- 用户能在 3 秒内理解产品的核心价值：好写、好找、能沉淀。
- 页面中明确出现“记录日常与成长”的表达，同时以克制方式带出“知识脉络 / 能力内化”的第二层价值。
- 桌面端与窄屏窗口下都能保持清楚层级与舒适阅读。
- 页面仍然保持轻量入口属性，不像营销页、登录页或复杂管理系统首页。
