## 1. 背景

当前编辑区已经具备右键菜单体系，也已经具备 Markdown 预览能力，但仍缺少一个高频写作场景中的基础动作：把本地图片直接插入到当前笔记。

现状问题主要有三点：

1. 用户无法在编辑区通过右键快速插入图片，只能依赖手写 Markdown 语法或手工处理文件路径。
2. 图片资源没有与知识库现有的隐藏 assets 目录预留设计打通，容易出现资源散落、引用不统一的问题。
3. 预览侧虽然支持 img 标签，但缺少本轮明确的图片展示约束，容易出现宽图溢出容器、只显示局部的问题。

用户已经确认当前知识库在创建时会预建一个隐藏的 assets 目录，且该目录默认不显示在左侧树中。这为统一图片资源存放位置提供了现成边界。

## 2. 目标

1. 在编辑区右键菜单中新增“插入图片”能力，降低写作时插图成本。
2. 所有通过该入口插入的图片统一复制到知识库根下隐藏的 notes/assets 目录。
3. 插入成功后立即把 Markdown 图片语法写入当前笔记，而不是推迟到保存阶段补偿。
4. 预览区正确显示图片，并确保图片不会横向溢出导致只显示一部分。
5. 延续当前产品边界，不让 assets 目录出现在左侧笔记树中。

## 3. 非目标

本轮不扩展以下范围：

- 通用附件系统。
- 图片压缩、裁剪、格式转换。
- 多图批量插入。
- 拖拽上传图片。
- 图片资源管理面板。
- 引用计数、垃圾图片回收。

## 4. 核心设计结论

### 4.1 采用专用图片插入链路

本轮采用专用的“插入图片”链路，而不是把需求泛化为通用资源系统。

原因：

- 用户当前需求明确聚焦图片。
- 现有右键菜单和预览链路已足够承载该能力。
- 专用链路可以在最小范围内闭环“选图、复制、写 Markdown、预览显示”。

### 4.2 资源统一落在知识库级 assets 目录

所有通过该入口插入的图片，都统一复制到知识库根下隐藏的 notes/assets 目录。

采用知识库级公共目录，而不是每篇笔记一个目录或每个笔记本一个目录，原因是用户已经明确该目录是预留设计的一部分，且当前产品约束要求继续保持其在左侧树中隐藏。

### 4.3 文件名由系统生成，不保留原名

图片复制后文件名由系统统一生成，采用“时间戳 + 短随机串 + 原扩展名”的方式，例如：

- 20260609-164512-a1b2c3.png

这样可以避免公共 assets 目录中的重名和语义污染。

### 4.4 Markdown 直接写相对路径

插入成功后，编辑器直接把图片 Markdown 语法写入正文，路径采用“相对当前笔记文件”的相对路径。

例如当前笔记位于 notes/项目A/周报.md，图片位于 notes/assets/20260609-164512-a1b2c3.png，则正文写入：

![图片](../assets/20260609-164512-a1b2c3.png)

这样在整个 notes 目录迁移时，图片引用仍然成立。

### 4.5 预览层只负责显示，不参与补偿写入

预览层不承担资源复制、路径修正或保存期补偿写入职责。资源复制和 Markdown 改写在用户触发插图时一次性完成。

这样可以避免以下不一致：

- Markdown 已写入但图片尚未复制。
- 预览可见但文件保存后失效。
- 保存失败后正文与实体文件状态分叉。

## 5. 交互设计

### 5.1 入口位置

“插入图片”进入现有编辑区右键菜单体系，包含两个入口：

- editor blank menu：编辑区空白态右键菜单。
- editor selection menu：编辑区有选区时的右键菜单。

两处都使用同一个动作 id 和同一条执行逻辑。

### 5.2 执行动作

用户点击“插入图片”后：

1. 前端触发新的菜单动作。
2. 动作调用新的 Tauri 命令。
3. Tauri 命令打开系统文件选择器，限制为常见图片类型。
4. 命令把图片复制到 notes/assets。
5. 命令返回当前笔记可直接引用的相对路径。
6. MarkdownEditor 把图片语法插入当前内容。

### 5.3 插入位置规则

如果当前有选区：

- 用图片语法整体替换选区内容。

如果当前没有选区：

- 在当前光标位置插入图片语法。

默认插入文本为：

- ![图片](相对路径)

本轮不增加额外弹窗让用户填写 alt 文本，以保证右键插图是一个低打断动作。

### 5.4 菜单可用性规则

如果当前没有打开任何笔记：

- “插入图片”在编辑区菜单中显示为禁用态。

原因是该动作依赖“当前笔记路径”来计算相对路径，没有当前笔记时无法落盘和计算引用。

## 6. 存储与路径规则

### 6.1 存储位置

图片统一复制到：

- notes/assets/

该目录继续保持隐藏，不进入左侧树展示。

### 6.2 文件命名规则

命名格式固定为：

- YYYYMMDD-HHMMSS-随机短串.原扩展名

示例：

- 20260609-164512-a1b2c3.png

约束：

- 保留原扩展名。
- 不保留原始文件名。
- 即使同一秒内连续插入，也依赖短随机串避免冲突。

### 6.3 相对路径计算规则

Tauri 命令返回的路径应当以“当前笔记所在目录”为基准进行相对化，而不是返回磁盘绝对路径。

原因：

- Markdown 文件应保持可迁移。
- 预览与编辑内容应避免平台绝对路径耦合。
- 相对路径与当前知识库内部资源模型更一致。

### 6.4 路径权威性

路径计算权威放在后端命令，而不是前端。前端只负责接收结果并写入 Markdown。

原因：

- 后端更接近真实文件系统状态。
- 可以统一处理路径规范化和平台差异。
- 可以避免前端重复实现路径运算细节。

## 7. 预览显示规则

### 7.1 图片样式约束

MarkdownPreview 对 img 统一施加以下显示约束：

- max-width: 100%
- height: auto
- display: block
- margin: 12px auto

必要时可补充：

- object-fit: contain

### 7.2 显示目标

目标不是把图片放大展示，而是保证：

1. 图片不会超出预览容器宽度。
2. 图片完整显示，不因容器裁切只展示局部。
3. 小图不被强制拉伸变糊。

### 7.3 协议边界

本轮不改变 Markdown 渲染协议，只做最小必要的路径放行和样式约束。

如果现有 sanitizer 或 URI 白名单对相对图片路径限制过严，则只做足以支持知识库内相对图片路径的最小调整，不扩展到任意本地绝对路径。

## 8. 错误处理

### 8.1 用户取消

如果用户在系统选图器中取消选择：

- 整个动作静默结束。
- 不写入任何 Markdown。
- 不弹错误提示。

### 8.2 非图片文件

如果用户选择的文件不属于允许的图片类型：

- 阻止复制。
- 不写入 Markdown。
- 向用户展示一次明确错误。

### 8.3 复制失败

如果复制到 notes/assets 失败，或路径计算失败：

- 不写入 Markdown。
- 向用户展示失败原因。

### 8.4 无当前笔记

如果当前没有打开笔记：

- 不允许执行插图动作。
- 通过禁用菜单项提前阻止，而不是等到执行时报错。

## 9. 实现边界

### 9.1 前端边界

前端改动集中在以下区域：

- 菜单动作 id 定义。
- 编辑区上下文菜单 schema。
- 菜单动作执行器。
- MarkdownEditor 中的图片插入执行逻辑。
- MarkdownPreview 中的图片显示样式约束。

前端不负责：

- 打开系统文件选择器。
- 文件复制。
- 最终路径规范化。

### 9.2 后端边界

后端新增一个专用 Tauri 命令，职责仅限：

1. 打开系统图片选择器。
2. 校验图片文件类型。
3. 复制到 notes/assets。
4. 返回可写入 Markdown 的相对路径。

该命令不负责：

- 修改编辑器正文。
- 更新索引数据库。
- 处理预览样式。

### 9.3 左侧树边界

本轮不改变左侧树展示逻辑，只要求继续隐藏 assets 目录。

## 10. 测试设计

本轮采用 TDD，最少覆盖以下测试：

### 10.1 菜单测试

1. editor blank menu 包含“插入图片”。
2. editor selection menu 包含“插入图片”。
3. 没有当前笔记时，“插入图片”为禁用态。

### 10.2 编辑器测试

1. 插入成功后，图片 Markdown 正确写入光标位置。
2. 有选区时，图片 Markdown 正确替换选区。
3. 用户取消选图时，编辑内容保持不变。
4. 后端返回失败时，编辑内容保持不变。

### 10.3 预览测试

1. 相对路径图片能渲染为 img。
2. 图片应用最大宽度约束，不发生横向溢出。

### 10.4 后端测试

1. 生成的文件路径位于 notes/assets。
2. 文件名符合“时间戳 + 随机串 + 扩展名”规则。
3. 返回路径相对当前笔记正确可用。
4. 非图片文件被拒绝。

## 11. 验收标准

满足以下条件即可认为本功能达标：

1. 用户可在编辑区右键菜单中看到“插入图片”。
2. 选图后图片被复制到隐藏的 notes/assets。
3. Markdown 正文立即写入图片语法。
4. 写入路径为相对当前笔记的相对路径。
5. 左侧树不展示 assets 目录。
6. 预览区能正确显示图片，且图片不会溢出容器导致只显示部分区域。
7. 用户取消或插入失败时，不污染正文内容。

## 12. 风险与处理

### 12.1 相对路径白名单风险

如果当前 MarkdownPreview 的 URI 白名单对相对路径图片支持不完整，插入后可能正文正确但预览不显示。

处理：

- 仅放开知识库内图片渲染所需的最小相对路径规则。

### 12.2 公共 assets 目录增长风险

随着图片增多，notes/assets 会持续膨胀，后续可能出现资源管理压力。

处理：

- 本轮先通过统一命名和固定目录保证一致性。
- 后续如需要，再单独设计图片资源管理与清理策略。

### 12.3 光标插入行为差异风险

不同选区状态下，如果插入逻辑没有统一实现，容易出现光标错位或替换范围错误。

处理：

- 在 MarkdownEditor 中只保留一条权威插入函数。
- 用测试覆盖“空光标”和“有选区”两种情况。
