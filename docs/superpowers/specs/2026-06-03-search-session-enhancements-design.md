## 1. 背景

上一轮搜索改造已经解决了三个核心问题：

- 全文搜索结果改为命中级结果，而不是一篇文章只返回一条。
- 点击结果后，编辑区和预览区都能同步滚动到命中位置。
- 搜索结果已经按相关度稳定排序，并支持同一篇文章多条命中逐条展示。

但从实际体验看，搜索仍然缺少完整的“搜索会话”概念：

- 搜索框没有历史，用户无法快速复用最近搜过的关键词。
- 搜索结果跳转后只有滚动，没有明确的当前命中高亮反馈。
- 用户进入文章后不能继续沿当前结果集查看上一个或下一个命中。

这会让搜索停留在“一次性跳转工具”的层级，而不是“阅读中的连续导航工具”。知识库场景下，用户经常需要围绕同一个关键词连续浏览多个命中，因此必须把搜索从“查一次”升级为“带状态的连续浏览会话”。

## 2. 目标

本轮设计目标如下：

- 为搜索框增加最近搜索词历史。
- 为搜索入口增加最近打开的搜索命中历史。
- 用户从搜索结果进入文章后，编辑区和预览区都必须高亮当前命中对象，而不是只滚动到附近。
- 建立搜索会话状态，使用户进入文章后可以继续查看当前结果集中的下一个或上一个命中。
- 提供紧凑、低干扰的搜索会话状态条，承载当前关键词、结果计数、导航动作与退出动作。
- 支持单字母快捷键：`N` 下一个、`B` 上一个、`Esc` 退出高亮与搜索会话。
- 快捷键行为不能破坏编辑区正常输入体验。

## 3. 非目标

本轮不做以下内容：

- 搜索语法增强，如布尔表达式、字段过滤、时间过滤。
- 搜索结果分组、折叠、分页和虚拟滚动优化。
- 搜索结果批量操作，如批量替换、批量标注。
- 跨知识库搜索或跨窗口同步搜索会话。
- 命中级高亮颜色自定义和用户偏好面板。

本轮重点是把现有命中级搜索补齐为可连续浏览的搜索会话。

## 4. 方案结论

本轮采用以下统一方案：

- 历史采用“两层模型”：最近搜索词 + 最近打开的搜索命中。
- 搜索跳转后的持续导航采用“搜索会话”模型，而不是让搜索弹层持续常驻。
- 搜索会话 UI 采用方案 C：工作区底部紧凑状态条。
- 上一个 / 下一个默认按当前整组搜索结果集导航，必要时允许跨文章切换。
- 快捷键采用单字母方案：`N` 下一个，`B` 上一个，`Esc` 退出会话。
- 快捷键只在“搜索会话激活且不处于普通文本输入态”时生效。

这样可以保留搜索框作为“查找入口”的职责，同时把进入文章后的连续浏览职责收敛到工作区本身。

## 5. 产品定义

本轮将搜索进一步定义为三段式流程：

1. 搜索框负责发起查询、展示结果、复用历史。
2. 点击命中后进入文章，系统创建一段搜索会话。
3. 搜索会话负责当前命中高亮、全局结果集前后切换和退出。

因此“搜索结果”与“搜索会话”是两个相关但不同的对象：

- 搜索结果：某个关键词查出来的一组命中项。
- 搜索会话：用户从这组结果中选中某一项后，在阅读区延续浏览这组结果的状态。

## 6. 交互设计

### 6.1 搜索框空态

当搜索框为空时，不触发搜索请求，而是展示两个区域：

- 最近搜索词。
- 最近打开的搜索命中。

展示顺序：

1. 最近搜索词。
2. 最近打开的搜索命中。

原因：用户大多数时候先复用关键词，再决定是否回到某个历史命中。

### 6.2 搜索结果态

输入非空查询后，搜索框继续展示命中级结果列表，保留当前已有能力：

- 300ms 防抖。
- 相关度排序。
- 键盘上下选择。
- 回车打开。
- 鼠标点击打开。

结果项结构保持命中级展示，但可补充更明确的“当前是标题命中还是正文命中”“第 N 行”等辅助信息。

### 6.3 进入文章后的搜索会话

用户从搜索结果进入文章后，系统创建一个搜索会话，并在工作区底部显示紧凑状态条。状态条至少包含：

- 当前关键词。
- 当前命中序号与总命中数，如 `2 / 17`。
- 上一个按钮。
- 下一个按钮。
- 退出提示或关闭动作。

状态条是会话型 UI，不要求永久存在；只有当搜索会话激活时显示。

### 6.4 上一个 / 下一个行为

默认导航范围是当前搜索结果全集，而不是仅限当前文章。

具体行为：

- 当前文章内存在后续命中时，直接切到当前文章中的下一个命中。
- 当前文章内没有后续命中时，自动打开下一条结果对应的文章，并定位到它的命中位置。
- 上一个行为同理，允许跨文章回退。

这保证“全局搜索”的语义和体验一致，不要求用户每切一篇文章都重新打开搜索框。

### 6.5 退出行为

按 `Esc` 或点击状态条关闭动作后：

- 清除当前搜索会话。
- 清除 editor 和 preview 中当前搜索高亮。
- 关闭底部状态条。
- 不清除历史记录。

退出只结束“会话”，不删除“历史”。

## 7. 快捷键与输入冲突规则

### 7.1 快捷键定义

在搜索会话激活时：

- `N`：跳到下一个命中。
- `B`：跳到上一个命中。
- `Esc`：退出当前搜索会话与高亮。

### 7.2 生效边界

这些快捷键不是全局无条件拦截，而是只在“搜索会话激活且不处于普通文本输入态”时接管。

允许生效的场景：

- 预览区处于活动态。
- 底部搜索会话状态条处于活动态。
- 编辑区存在搜索会话，但当前不在普通文本录入态。

禁止生效的场景：

- 编辑区正在进行正常文本输入。
- 输入框、文本域或其他可编辑控件正在接收普通字符输入。

原因：若在编辑区正常输入时直接拦截 `N` / `B`，会破坏英文输入与普通编辑体验。

### 7.3 输入态判定

实现层面不要求复杂的焦点系统，但必须满足以下原则：

- 当事件来源是普通输入控件时，不拦截 `N` / `B`。
- 当 CodeMirror 处于常规编辑输入状态时，不拦截 `N` / `B`。
- `Esc` 可以优先结束搜索会话，但不能破坏已存在的必要编辑器默认行为；若同一时刻只需要搜索会话退出，则优先退出搜索会话。

## 8. 前端状态设计

现有前端已经有 `searchNavigationTarget`，本轮需要在其上补一层搜索会话状态。

建议新增搜索会话状态结构：

```ts
interface SearchSessionState {
  query: string;
  results: SearchResult[];
  currentIndex: number;
  active: boolean;
}
```

职责：

- `query`：当前搜索会话关键词，用于状态条展示。
- `results`：当前会话绑定的完整结果集，用于前后切换。
- `currentIndex`：当前命中在结果集中的索引。
- `active`：当前是否存在搜索会话。

状态来源：

- 当用户从搜索弹层打开某条结果时，写入完整的 `results`、`query` 和 `currentIndex`。
- 当用户通过 `N` / `B` 或状态条按钮切换时，只更新 `currentIndex`，并派生新的 `searchNavigationTarget`。
- 当用户按 `Esc` 退出时，清空该会话状态。

这样 `searchNavigationTarget` 继续负责“当前要跳去哪里”，而 `SearchSessionState` 负责“当前这一轮搜索还能怎么继续走”。

## 9. 渲染与高亮设计

### 9.1 编辑区高亮

编辑区不再仅滚动到目标行，还要对当前搜索命中建立显式高亮。

高亮要求：

- 优先高亮当前命中的精确文本范围。
- 当无法在该行精确恢复唯一范围时，至少高亮目标行中的匹配文本。
- 同一时刻只强调当前命中，不同时渲染整篇文章所有搜索结果高亮。

原因：本轮关注的是“当前命中确认感”，不是全文搜索词批量染色。

### 9.2 预览区高亮

预览区也必须对当前命中做显式高亮，而不是只滚动。

要求：

- 继续基于已有 source-line 翻译机制定位到对应元素。
- front matter 被剥离时，仍要先做行号平移再判断目标位置。
- 若目标命中完全落在被剥离的 front matter 区域，则不显示正文高亮，并保持当前已存在的安全兜底行为。

### 9.3 高亮更新规则

当以下任一事件发生时，应更新 editor 和 preview 高亮：

- 从搜索结果打开命中。
- 通过状态条点击上一个 / 下一个。
- 通过快捷键切换上一个 / 下一个。

当退出搜索会话时，应同时移除两边高亮。

## 10. 历史记录策略

### 10.1 最近搜索词

最近搜索词记录规则：

- 用户触发有效搜索后写入。
- 空字符串不写入。
- 重复关键词去重并移动到顶部。
- 仅保留有限条目，建议 8 到 10 条。

### 10.2 最近打开的搜索命中

最近打开的搜索命中记录规则：

- 用户真正打开某条命中后写入。
- 记录至少包含关键词、文章标题、文章路径、命中片段、命中位置。
- 相同命中再次打开时移动到顶部而不是重复新增。
- 仅保留有限条目，建议 8 到 10 条。

### 10.3 存储层选择

本轮优先使用前端本地持久化即可，不要求进入后端数据库。

原因：

- 历史是偏个人、偏界面层的状态。
- 本轮目标是补齐体验，不需要跨设备同步。
- 本地持久化更轻量，改动面更小。

## 11. 测试策略

本轮至少覆盖以下测试：

- 搜索框空态展示最近搜索词与最近打开命中。
- 用户打开搜索结果后，正确写入搜索会话状态。
- editor 在收到搜索导航目标后不仅滚动，而且显示当前命中高亮。
- preview 在收到搜索导航目标后不仅滚动，而且显示当前命中高亮。
- `N` / `B` 在搜索会话激活且非普通输入态时能切换结果。
- `N` / `B` 在普通文本输入态不抢占字母输入。
- `Esc` 结束搜索会话并清除高亮。
- 当前文章切到边界后，继续切换会自动进入下一篇或上一条结果对应的文章。

优先做前端窄测，不要求本轮新增 e2e 才能交付，但若已有可复用路径，后续可以补充一条会话导航 e2e。

## 12. 风险与后续扩展

### 12.1 主要风险

- 单字母快捷键与编辑输入冲突，需要边界判定清晰。
- preview 精确高亮可能受 Markdown 渲染结果影响，需要允许“无法精确到字符时回退到行内匹配”。
- 历史持久化若直接耦合在弹层组件内部，后续会难以复用，应尽量抽到清晰状态层。

### 12.2 后续扩展

本轮完成后，可继续扩展：

- 搜索会话状态条上增加“回到结果列表”。
- 支持仅当前文章内导航与全局结果集导航的显式切换。
- 搜索历史支持删除单条和清空。
- 搜索命中高亮支持更明显的动画反馈或主题定制。

但这些都不应阻塞本轮目标。
