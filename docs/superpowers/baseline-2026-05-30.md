## 1. Baseline 结论

当前仓库可作为 Phase 2 之后继续开发的 baseline。

本次 baseline 前已经处理掉审查中发现的阻断项：

- watcher 与 note service 的锁顺序反转风险。
- 知识库相对路径 `../` 逃逸风险。
- watcher 对删除/重命名文件时索引陈旧的问题。
- 搜索 LIKE fallback 未转义 `%`、`_`、`\\` 的问题。
- 导入完成后异步打开笔记未等待的问题。
- 临时便签文件 `docs/test.md` 含本机绝对图片路径的问题。

剩余问题不会阻断继续开发，但必须纳入后续任务队列，避免在后续功能上叠加技术债。

## 7. 后续开发流程约束

后续项目开发必须按 superpowers 方法推进：

1. 开始新功能或行为修改前，先使用适用的 superpowers 技能。
2. Bug 或异常行为先走 systematic debugging，不直接猜修。
3. 功能或 bugfix 默认先写测试或至少明确可验证用例。
4. 多步骤任务先写计划，再按计划执行。
5. 每个重要阶段完成后运行验证命令，不用“应该可以”替代证据。
6. 重大变更提交前必须 code review，Critical / Important 反馈先处理。
7. 提交前至少运行：

```bash
cd /Users/lijun/mynote/src-tauri && cargo test
cd /Users/lijun/mynote && corepack pnpm build
```

8. 需要手动验证 UI 时，使用开发命令：

```bash
export PATH="$HOME/.npm-global/bin:$PATH"
pnpm tauri dev
```

或：

```bash
corepack pnpm tauri dev
```
